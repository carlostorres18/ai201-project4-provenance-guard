# Provenance Guard - Project Planning

## Detection Signals

### Signal 1: Linguistic Style Analysis

**What it measures**

This signal measures writing style characteristics, including:

* Average sentence length
* Sentence length variation
* Vocabulary diversity (type-token ratio)
* Word repetition
* Readability score

**Output**

A normalized score between **0.0 and 1.0**.

* 0.0 = strongly human-like style
* 1.0 = strongly AI-like style

**Why it works**

Human writing naturally contains more variation in sentence structure, vocabulary, and rhythm. AI-generated text often exhibits smoother, more uniform writing patterns.

**Blind spots**

This signal performs poorly on:

* Professional human writing with highly polished grammar
* AI-generated text that has been heavily edited
* Very short text samples

---

### Signal 2: Perplexity Analysis

**What it measures**

This signal measures how predictable the submitted text is according to a language model.

Lower perplexity indicates the text is statistically predictable, while higher perplexity indicates greater linguistic variation.

**Output**

A normalized score between **0.0 and 1.0**.

* 0.0 = strongly human-like
* 1.0 = strongly AI-like

**Why it works**

AI systems generate highly probable word sequences. Human writing often contains more surprising wording and creative phrasing.

**Blind spots**

This signal performs poorly on:

* Technical documentation
* Children's writing
* AI text rewritten by humans
* Very creative AI-generated writing

---

### Combining Signals

Both signals are weighted equally.

```
confidence = (style_score + perplexity_score) / 2
```

Final confidence is clamped between **0.0 and 1.0**.

The attribution result is determined using the confidence thresholds described below.

---

## Uncertainty Representation

The confidence score represents how strongly the combined evidence supports an AI attribution.

Example:

* **0.60** means the evidence slightly favors AI, but not enough for a high-confidence decision.
* **0.95** means both detection signals strongly indicate AI-generated content.
* **0.10** means both signals strongly favor human authorship.

### Confidence Thresholds

| Confidence | Classification |
| ---------- | -------------- |
| 0.00–0.39  | Likely Human   |
| 0.40–0.60  | Uncertain      |
| 0.61–1.00  | Likely AI      |

Using an uncertainty range prevents weak evidence from producing overly confident labels.

## Transparency Label Design

### High-Confidence AI

**AI-Generated (High Confidence)**

> Our system found strong evidence that this content was generated using artificial intelligence. Confidence: High.

---

### High-Confidence Human

**Human-Written (High Confidence)**

> Our system found strong evidence that this content was written by a human author. Confidence: High.

---

### Uncertain

**Attribution Uncertain**

> Our system found mixed evidence and cannot confidently determine whether this content was written by a human or generated by AI.

## Appeals Workflow

### Who can appeal?

Any creator whose content has received an attribution decision.

### Information submitted

* Content ID
* Creator explanation
* Timestamp

Example:

```
{
    "content_id": "12345",
    "reason": "I wrote this article myself."
}
```

### What happens?

When an appeal is submitted:

1. The appeal is stored.
2. The original audit log entry is updated.
3. The content status changes to:

```
Under Review
```

4. A new audit log entry records the appeal.

### Human Reviewer Queue

A reviewer should see:

* Content ID
* Original text
* Original classification
* Confidence score
* Individual signal scores
* Creator explanation
* Submission timestamps
* Current review status

---

## Anticipated Edge Cases

### 1. Poetry

Poetry often uses:

* repetitive wording
* simple vocabulary
* unusual sentence structure

These characteristics may resemble AI-generated text and produce a false positive.

---

### 2. Technical Documentation

Technical documents frequently contain repetitive terminology and consistent sentence structures.

The detector may incorrectly classify them as AI-generated.

---

### 3. Edited AI Content

A human may substantially edit AI-generated content.

The writing style becomes more human-like while perplexity may still appear AI-like, resulting in an uncertain classification.

---

### 4. Very Short Submissions

A short paragraph or a few sentences provide insufficient information for reliable stylistic analysis.

The system will likely return an uncertain result.

# Architecture

## Submission Flow

```
                    Raw Text
                       │
                       ▼
                 POST /submit
                       │
                       ▼
                 Rate Limiter
                       │
                       ▼
         Linguistic Style Analyzer
             (style score)
                       │
                       ▼
            Perplexity Analyzer
          (perplexity score)
                       │
                       ▼
            Confidence Scorer
                       │
                       ▼
      Transparency Label Generator
                       │
                       ▼
                Audit Logger
                       │
                       ▼
                JSON Response
```

### Appeal Flow

```
              POST /appeal
                     │
                     ▼
             Appeal Validator
                     │
                     ▼
              Status Updater
            (Under Review)
                     │
                     ▼
               Audit Logger
                     │
                     ▼
              JSON Response
```

### Architecture Narrative

When a creator submits text, the request passes through the submission endpoint and rate limiter before entering the two-signal detection pipeline. The style analyzer and perplexity analyzer each produce a normalized score, which the confidence scorer combines into a single confidence value. The transparency label generator selects the appropriate user-facing label, the decision is stored in the audit log, and the API returns the classification, confidence score, and label.

If a creator disputes the result, they submit an appeal through the `/appeal` endpoint. The system records the creator's explanation, updates the content status to **Under Review**, logs the appeal alongside the original decision, and returns a confirmation response without automatically changing the original classification.
